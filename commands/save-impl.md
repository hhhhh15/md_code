1. 如果是从 /save-read 跳转过来，直接使用已知的类名、日期、功能名称

2. 如果是单独执行，询问用户：类名是什么？对应哪个功能？

3. 扫描对应的 `读懂-{类名}-{日期}.md`：
   - 从 `# ✅ 实现的功能` 区获取功能列表
   - 从 `# ❓ 不懂的代码` 区匹配某个功能的相关问题
   - 判断闭环标准：该功能相关问题中，有某个"我的理解"已完整覆盖整体流程（不要求每题都填）
   - 只对已闭环的功能生成实现块

4. 检查 `contexts/` 下是否已存在 `实现-{类名}-{优先级}-{日期}.md`
   - 存在：在文件末尾追加新的实现块
   - 不存在：新建文件，使用完整模板

5. 文件保存到 `contexts/实现-{类名}-{优先级}-{日期}.md`
