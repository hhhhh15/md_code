⚠️ 节省 token 的最优工作流

- 有闭环功能则再执行 /save-impl → 生成实现块
- 全程只有两次文件写入，已存在文件只追加不重读,可重建

---

当用户执行 /save-read 时，按以下步骤执行：

0. 启动时先扫描 contexts/ 目录
    找最近一个 `读懂-*` 文件，
    读取内容加载为上下文
    告诉用户：「已加载：读懂-xxx-2025-01-13.md」

1. 询问用户：类名是什么？当前分支是什么？

2. 检查 contexts/ 目录是否已存在同类名文件
    ├─ 存在 → 询问用户：「已有该文件，是否重新创建？」
    │   ├─ 是 → 继续走创建流程
    │   └─ 否 → 判断本次对话内容
    │       ├─ 有新问题/不懂的代码 → 直接追加到 # ❓ 不懂的代码 末尾，不读其他部分
    │       └─ 用户指定其他区域 → 定向更新对应区域
    └─ 不存在 → 继续走创建流程

3. 新建文件需要把模板中的所有一级标题全部渲染
4. 获取今天日期，格式 YYYY-MM-DD
5. 生成文件保存到 `contexts/读懂-{类名}-{优先级}-{日期}.md`

## 表格生成规则
生成任何表格时：
1. 先找出每列最长的单元格字符数
2. 所有行该列宽度 = 最长字符数 + 2个空格padding
3. 中文字符算2个字符宽度
4. 内容超过20字的列，宽度参考第二长的，超出内容换行显示
5. 分隔行 `|---|` 也要对应等宽
