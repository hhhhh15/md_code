### Playwright 核心用法（直接复制自对话）

**它是什么 / 怎么工作的**

Playwright 通过 **Chrome DevTools Protocol (CDP)** 控制浏览器进程，本质是你的 Python 脚本 → WebSocket → 浏览器进程，浏览器真实执行操作。

```
你的脚本 → Playwright Python → WebSocket → Chromium/Firefox/WebKit
```

不是截图模拟点击，是真实的浏览器，JS 会执行，Cookie 会存，和真人用浏览器没区别。

**三个核心对象**

```
Browser → BrowserContext → Page
```

- `Browser`：浏览器进程本身
- `BrowserContext`：独立的会话（独立 Cookie、缓存），相当于一个"用户身份"
- `Page`：一个标签页

```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)  # headless=False 可以看到窗口
    context = browser.new_context()
    page = context.new_page()
    page.goto("https://www.xiaohongshu.com")
    browser.close()
```

**手机模拟模式**

```python
with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)
    iphone = p.devices["iPhone 12"]
    context = browser.new_context(**iphone)
    # iphone 里包含了：UA、屏幕尺寸、触摸支持、像素比
    page = context.new_page()
    page.goto("https://www.xiaohongshu.com")
```

**保存 / 加载登录状态（关键）**

```python
# 第一次：手动登录，保存 Cookie
with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)
    context = browser.new_context(**p.devices["iPhone 12"])
    page = context.new_page()
    page.goto("https://www.xiaohongshu.com")
    input("请手动登录，登录完成后按回车...")
    context.storage_state(path="auth.json")  # 保存整个会话状态
    browser.close()

# 之后每次：直接加载
with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)
    context = browser.new_context(
        **p.devices["iPhone 12"],
        storage_state="auth.json"  # 加载登录态
    )
    page = context.new_page()
    page.goto("https://www.xiaohongshu.com")
    # 已经是登录状态了
```

**常用操作**

```python
page.goto("https://...")
page.wait_for_selector(".note-item")       # 等待元素出现再操作（重要，不要用 sleep）
page.click(".note-item:first-child")
page.evaluate("window.scrollBy(0, 800)")   # 滚动
text = page.inner_text(".content")         # 获取文本
src = page.get_attribute("img", "src")     # 获取属性（比如图片 src）
page.screenshot(path="page.png")
print(page.url)
```

## 补充知识点

知识点：是什么 | 怎么用/步骤

1. Playwright：Python 浏览器自动化库，通过 CDP 协议控制 Chromium，不是截图模拟，是真实浏览器操作 | pip install playwright → playwright install chromium → 写脚本调用

2. Chromium：Google 开源的浏览器内核，Chrome 基于它做的。Chromium = 开源裸浏览器，Chrome = Chromium + Google 私有功能。Playwright 用 Chromium 因为它开源可被程序直接控制 | playwright install chromium 自动下载，不需要手动装 Chrome

3. CDP（Chrome DevTools Protocol）：Playwright 与浏览器通信的底层协议，走 WebSocket | 不需要手动操作，Playwright 内部自动使用

4. storage_state：Playwright 保存完整会话状态（Cookie + localStorage）的方式，比单独保存 Cookie 更完整 | context.storage_state(path="auth.json") 保存，new_context(storage_state="auth.json") 加载

5. 虚拟环境 venv：隔离项目依赖，避免和其他项目冲突 | python -m venv venv → venv\Scripts\activate（Windows）

6. headless 模式：headless=False 可以看到浏览器窗口，True 则后台静默运行 | p.chromium.launch(headless=False)

7. MCP（Model Context Protocol）：让 AI 直接控制外部工具的协议。playwright-mcp 是微软官方出的，把 Playwright 包装成 MCP Server，这样 AI 可以直接控制浏览器。和自己写脚本不同 | 本项目选择自己写脚本，不用 MCP

8. OCR：图片文字识别，小红书帖子大量内容是图片而非文字，需要 OCR 才能提取步骤 | pytesseract 或 paddleocr，步骤7用到

## 延伸思考

- 程序运行期间电脑需开着；如需定时执行可用 Windows 任务计划程序；完全不依赖本机需租 VPS
- 想在手机上看脚本运行画面：装 ToDesk，headless=False，手机远程连电脑即可
- pip install 不需要从 GitHub 下载源码，直接从 PyPI 拉包
- 小红书移动端反爬相对弱，模拟手机 UA 是关键
- 官方 API 不对个人开放，只能走浏览器自动化
- 帖子内容大量是图片而非文字，OCR 是必要环节，会增加复杂度
- 频率控制很重要，需模拟人工操作节奏避免封号
